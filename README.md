# MGPUPlus

**MGPUPlus** — это iOS-приложение для студентов МГПУ, которое объединяет в одном месте персональное расписание, новости университета и личные заметки по учебе. Проект решает понятную повседневную задачу: студенту обычно приходится переключаться между несколькими источниками, чтобы посмотреть пары, узнать об изменениях и не забыть свои дела. В этом приложении я собрал ключевые учебные сценарии в одном мобильном интерфейсе.

Для меня этот проект также стал практикой работы с современным стеком Apple. Приложение построено на `SwiftUI`, использует `SwiftData` для локального хранения данных, `URLSession` для сетевых запросов и локализацию на русский и английский языки. Сейчас новости загружаются с сайта МГПУ, а расписание синхронизируется через локальный mock-источник и сохраняется в локальную базу. При этом архитектура уже подготовлена к переходу на реальный API через абстракцию `ScheduleDataSource`.

## Цели проекта

- Дать студенту быстрый доступ к персонализированному учебному экрану после выбора института и группы.
- Показывать расписание по дням с понятными статусами занятий: идет сейчас, скоро начнется, завершено, отменено, замена, онлайн.
- Объединить официальные новости университета и личные заметки пользователя в одном приложении.
- Отработать подход к созданию iOS-приложения с онбордингом, локальным хранилищем, асинхронной загрузкой данных, настройками и локализацией.

## Основные возможности

- Онбординг с выбором института и учебной группы.
- Три основные вкладки: новости, расписание, настройки.
- Расписание на день с фильтрацией по выбранной группе.
- Личные заметки и задачи на выбранную дату: создание, отметка о выполнении, удаление.
- Лента новостей с пагинацией, обновлением по свайпу и экраном подробной статьи.
- Переключение темы оформления: `system`, `light`, `dark`.
- Переключение языка приложения: `ru`, `en`.
- Локальное хранение данных через `SwiftData`.

## Архитектура и источники данных

- `ContentView` определяет, показывать ли онбординг или основной интерфейс, исходя из сохраненного учебного профиля.
- `OnboardingView` сохраняет выбранные институт и группу через `@AppStorage`.
- `Home` отображает недельный календарь и запускает синхронизацию расписания для выбранного дня.
- `ScheduleSyncService` синхронизирует внешние данные расписания с локальным хранилищем `SwiftData`.
- `MockScheduleDataSource` пока получает расписание из файла `schedule_mock.json`.
- `NewsService` загружает и парсит новости и страницы статей с сайта МГПУ.

Сейчас в приложении используются два разных подхода к данным:

- Новости: реальная загрузка по сети с сайта МГПУ.
- Расписание: локальный mock JSON через заменяемую абстракцию источника данных.

## Выводы по проекту

- Даже сравнительно небольшое студенческое приложение становится заметно удобнее, если разделить его на понятные сценарии: онбординг, контентные модули и настройки.
- `SwiftData` хорошо подходит для сочетания пользовательских данных, таких как заметки, и синхронизируемых сущностей, таких как расписание.
- Даже если реальный API еще не готов, имеет смысл заранее проектировать сервисный слой через протоколы и отдельные источники данных, чтобы потом заменить mock на production-реализацию без переписывания интерфейса.
- Локализация и поддержка светлой/темной темы заметно повышают ощущение завершенности продукта даже на раннем этапе.

## Инструкция по запуску

1. Клонировать репозиторий:

```bash
git clone https://github.com/DmitryBeton/MCU-Plus.git
```

2. Открыть проект в Xcode:

```bash
open MGPUPlus.xcodeproj
```

3. Выбрать схему `MGPUPlus`.
4. Выбрать iOS Simulator с `iOS 18.0` или новее.
5. Запустить приложение через `Cmd + R`.

## Текущие ограничения

- Расписание пока не загружается из реального API университета.
- Новости парсятся из HTML-страниц сайта, поэтому изменения в верстке сайта могут повлиять на корректность загрузки.
- Список институтов и групп сейчас захардкожен внутри приложения.
- В проекте пока нет автоматических тестов.

## Планы по доработке

- Заменить `MockScheduleDataSource` на реализацию, работающую с API, чтобы приложение показывало актуальное расписание выбранной группы, а не данные из mock-файла.
- Перевести загрузку новостей с HTML-парсинга на официальный API или структурированный endpoint, чтобы уменьшить зависимость от изменений разметки сайта МГПУ.
- Вынести сетевой слой в более явную архитектуру с моделями запросов и ответов, обработкой ошибок и кэшированием, чтобы сделать загрузку данных надежнее и удобнее в поддержке.
- Перенести справочник институтов и групп из кода во внешний источник или API, чтобы экран онбординга можно было масштабировать без ручного обновления `StudyCatalog`.
- Добавить тесты для парсинга новостей, синхронизации расписания и логики выбора учебного профиля, чтобы снизить риск регрессий.
- Добавить уведомления об изменениях в расписании, чтобы студент быстрее узнавал о заменах, отменах и смене формата занятий.
